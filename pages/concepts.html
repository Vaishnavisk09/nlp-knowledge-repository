Feature Engineering

Bag of Words (BoW)

Definition: Represents text using word frequencies.

Purpose: Converts text into numerical vectors.

Example: "I love NLP" → [I=1, Love=1, NLP=1]

Advantages:

  • Simple
  • Easy to implement

Limitation: Ignores word order.

N-Grams

Definition: Represents sequences of words.

Examples: Unigram Bigram Trigram

Application: Language prediction.

TF-IDF

Definition: Measures the importance of words.

Benefit: Highlights meaningful keywords.

Language Representation

One-Hot Encoding

Definition: Represents every word using binary vectors.

Advantage: Simple representation.

Limitation: Cannot capture semantic similarity.

Word2Vec

Definition: Learns dense vector representations of words based on context.

Advantages:

  • Captures semantic relationships.
  • Efficient for similarity tasks.

FastText

Definition: Represents words using character n-grams.

Benefit: Works well with rare and misspelled words.

Contextual Embeddings

Definition: Word meaning changes depending on surrounding context.

Examples: BERT GPT RoBERTa

Benefit: Highlights meaningful keywords.

Text Preprocessing

Sentence Segmentation

Definition: Splits text into individual sentences.

Purpose: Makes long text easier to process.

Example: "Hello. How are you?" → Two sentences.

Advantages:

  • Improves document understanding.
  • Useful for summarization.

Applications: Chatbots, translation, summarization.

Tokenization

Definition: Splits sentences into words or tokens.

Purpose: First step before feature extraction.

Example: "I love NLP" ↓ [I, love, NLP]

Advantages:

  • Easy text processing
  • Foundation for NLP

Applications: Search engines Text analytics

Case Normalization

Definition: Converts all words to lowercase.

Example: APPLE → apple

Advantage: Reduces duplicate words.

Stop-word Removal

Definition: Removes common words like "the", "is", "and".

Benefit: Improves processing speed.
